# Licensing and Open Source Release Guidance

## Release of Microsoft Content under CC BY 4.0 License

Follow the steps below to prepare and release your project as open source.

---

## 1. Required Files in the Project Root Directory

Ensure the following files are present in the root of your repository:

### **LICENSE-ASSETS**
For documentation, images, and other non-code content:
- Add a copy of the [CC BY 4.0 License](https://creativecommons.org/licenses/by/4.0/legalcode)
- Save it as `LICENSE-ASSETS`

---

### **README.md**
Your README must include:
- A link to the [Microsoft Open Source Code of Conduct](https://opensource.microsoft.com/codeofconduct)
- A [Trademark Notice](https://docs.opensource.microsoft.com/content/releasing/index.html)
- [Security Reporting Instructions](https://docs.opensource.microsoft.com/content/releasing/security.html)

Optional:
- Include additional project details using the [Release Checklist](https://docs.opensource.microsoft.com/releasing/release-checklist.html)

---

### **CONTRIBUTING.md**
If accepting external contributions:
- Provide clear contribution guidelines
- Include minimum guidance as described [here](https://docs.opensource.microsoft.com/releasing/overview.html#contribution-guidance)
- All contributions must comply with the Microsoft Contributor License Agreement (CLA)

📌 Note:
- GitHub repos automatically enforce CLA
- For non-GitHub projects, ask contributors to sign the [Microsoft CLA](https://cla.opensource.microsoft.com/)

---

### **Remove Sensitive Information**

Ensure the following are **NOT included** in your repository:

- Microsoft trademarks (unless approved)
- Microsoft product icons
- Internal-only comments or notes

---

## 2. Naming Compliance

Ensure the repository name follows the official:
- [Microsoft OSS Naming Process](https://docs.opensource.microsoft.com/content/releasing/brand.html)

---

## 3. Publishing Requirements

When releasing your project:
- Use a **Microsoft GitHub organization account**
- Do **NOT** publish from a personal account

Reference:
- [Open Source Release Guidance](https://docs.opensource.microsoft.com/releasing/overview.html)
