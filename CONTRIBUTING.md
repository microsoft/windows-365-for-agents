# Contributing to Windows 365 for Agents

Welcome, and thank you for your interest in contributing! There are many ways to contribute, and we appreciate all of them.

## How to Contribute

### Reporting Issues

Found a bug or have a feature request? [Open an issue](../../issues/new) with as much detail as possible:

- Steps to reproduce (for bugs)
- Expected vs. actual behavior
- Environment details (OS, .NET version, Python version, Agent 365 SDK version — whichever applies)
- Relevant logs or screenshots

### Suggesting Improvements

We welcome documentation improvements, code examples, and other enhancements. Please open an issue first to discuss the change before submitting a pull request.

### Pull Requests

1. **Fork** the repository
2. **Create a branch** from `main` for your changes
3. **Make your changes** with clear, descriptive commit messages
4. **Test** your changes
5. **Submit a pull request** with a clear description of what you changed and why

### Documentation Contributions

Documentation improvements are especially welcome. When contributing to docs:

- Use clear, concise language
- Follow the existing style and formatting
- Include code examples where helpful
- Test all code snippets

## Development Setup

```bash
# Clone the repo
git clone https://github.com/microsoft/windows-365-for-agents.git
cd windows-365-for-agents

# Install Python dependencies (for running examples)
pip install httpx
```
